# ID 기준 저장/연결 규칙

## 목적
이 시스템에서 ID가 존재하는 데이터는 항상 ID를 저장, 조회, 연결, 집계의 기준값으로 사용합니다.
이름이나 텍스트 컬럼은 화면 표시용 또는 스냅샷 용도로만 사용합니다.

## 우선 사용해야 하는 키
- `client_id`
- `product_id`
- `product_master_id`
- `document_item_id`
- `doc_id`
- `author_id`

## 기본 원칙
- 화면에 보이는 `client`, `product`, `name1`, `name2`, `author` 같은 문자열은 표시용으로만 사용합니다.
- 이미 ID가 있는 데이터를 저장하거나 수정할 때, 이름 문자열로 다시 찾거나 매칭하는 fallback 로직은 넣지 않습니다.
- 집계, 통계, 검색 조건도 가능하면 이름이 아니라 ID 기준으로 설계합니다.
- 저장 로직에서는 텍스트 기반 자동복구보다 ID 누락을 먼저 검증하고 막는 방향을 우선합니다.

## 구현 체크리스트
- 새로운 관계를 추가할 때 문자열 컬럼만으로 연결하지 않았는가
- API에서 저장 직전 ID 누락 검증이 있는가
- 화면에서 선택값은 이름이 아니라 ID를 내부 기준값으로 유지하는가
- 레거시 데이터 보정이 필요하면 런타임 fallback이 아니라 SQL 백필로 해결하는가

## 예외
- 직접입력처럼 마스터 ID가 없는 케이스는 명시적으로만 허용합니다.
- 직접입력 데이터는 일반 품목이나 거래처와 자동 혼합 매칭하지 않습니다.
