# 공통코드 / 마스터 / 직접입력 경계 규칙

## 목적
분류값, 기준값, 실데이터, 예외 입력을 서로 다른 층으로 관리합니다.

## 공통코드로 관리할 것
- 고정된 분류값
- 구분값
- 사용 여부와 정렬 순서가 필요한 값

예:
- 품목 구분
- 수신처 유형
- 상태 코드

원칙:
- DB에는 코드값을 저장합니다.
- 화면에는 코드명을 보여줍니다.

## 마스터 데이터로 관리할 것
- 실제 거래처
- 실제 품목
- 실제 공급처
- 실제 수신 대상 목록

원칙:
- 실데이터는 공통코드에 넣지 않습니다.
- 실데이터는 고유 ID를 가지는 별도 테이블로 관리합니다.

## 직접입력으로 허용할 것
- 마스터에 없는 예외 입력
- 임시 입력이지만 업무상 허용이 필요한 값

원칙:
- 직접입력은 명시적으로 구분합니다.
- 일반 마스터 데이터와 자동 매칭하거나 자동 집계 기준으로 섞지 않습니다.
- 직접입력은 운영상 예외라는 사실이 드러나야 합니다.
