# 품목 구조 규칙

## 목적
품목은 공통 품목 기준과 거래처별 품목 운영값을 분리해서 관리합니다.

## 역할 구분
- `product_masters`
  공통 품목 기준
- `products`
  거래처별 품목 조건과 단가

## product_masters 역할
- 같은 품목명을 하나로 묶는 상위 기준
- 견적, 통계, 대시보드에서 묶음 기준으로 사용
- 공통 품목명, 공통 기본명, 공통 분류를 가짐

## products 역할
- 거래처별 품목 운영 데이터
- `client_id` 기준으로 연결
- 거래처별 단가, 공급처, 예외 표기, 수신처 같은 하위 정보를 가짐
- 반드시 `product_master_id`로 공통 품목에 연결되는 방향을 유지

## 개발 시 주의
- 문서작성에서 품목 목록은 여전히 거래처별 품목(`products`) 기준으로 보여줍니다.
- 통계나 공통 집계는 가능하면 `product_masters` 기준으로 설계합니다.
- 공통 품목과 거래처별 품목 역할을 화면이나 API에서 다시 섞지 않습니다.
- 자동 백필은 출발점일 뿐이며, 이름을 다르게 적어둔 품목은 수동 검수 대상이 될 수 있습니다.
